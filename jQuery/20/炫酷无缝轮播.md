# 炫酷无缝轮播：从结构到交互的一次完整实现拆解

本文把“炫酷 + 无缝循环 + 可拖拽 + 自动播放 + 进度条 + 可访问性”这套轮播的设计思路按步骤拆开讲清楚，并在每一步提炼关键知识点。

---

## 0. 目标与约束

我们希望轮播具备：

- **视觉**：居中卡片更“立体”，左右卡片弱化；背景有随切换变化的氛围光。
- **交互**：按钮切换、指示点跳转、键盘左右键、拖拽/触控滑动。
- **体验**：自动播放 + 进度条；鼠标悬停暂停；切后台暂停；尊重“减少动态效果”偏好。
- **工程**：尽量少抖动、少重排；结构清晰，状态可控。

---

## 1. 第一步：搭建“轨道式”轮播结构

轮播最稳定的一类实现，是“**窗口 + 轨道**”模型：

- `.carousel__frame`：可视窗口（裁切区域）
- `.carousel__track`：轨道，横向摆放所有 slide
- `.carousel__slide`：每一张卡片
- 切换：只改 `track` 的 `transform: translate3d(x,0,0)`，移动整条轨道

对应的 HTML 结构（精简版）如下：

```html
<section class="carousel" id="coolCarousel" aria-roledescription="carousel" aria-label="炫酷轮播图" tabindex="0">
  <header class="carousel__header">
    <h1 class="carousel__title">jQuery 4.0 Carousel</h1>
    <p class="carousel__hint">拖拽 / 滑动 · 键盘 ← → · 自动播放</p>
  </header>

  <div class="carousel__viewport">
    <div class="carousel__frame" role="group" aria-label="轮播内容">
      <div class="carousel__track">
        <article class="carousel__slide" style="--bg: radial-gradient(...);">
          <div class="carousel__content">...</div>
        </article>
        ...
      </div>
    </div>
  </div>

  <footer class="carousel__controls" aria-label="轮播控制区">
    <div class="carousel__buttons">
      <button class="carousel__btn" type="button" data-action="prev" aria-label="上一张">←</button>
      <button class="carousel__btn" type="button" data-action="next" aria-label="下一张">→</button>
    </div>
    <div class="carousel__dots" role="tablist" aria-label="轮播指示点"></div>
    <div class="carousel__progress" aria-hidden="true"><div class="carousel__bar"></div></div>
  </footer>
</section>
```

**知识点重点**

- 轨道轮播的核心是“**位移**”，不是“显示/隐藏”。这样动画更连贯、实现更统一。
- `translate3d` 通常比改 `left` 更平滑，因为它更容易走合成层（GPU）。

## 2. 第二步：用 CSS 做“炫酷感”的两条主线

### 2.1 背景氛围光：跟随当前 slide 的渐变

容器的 `::before` 使用 `--active-bg` 作为背景，然后 blur 成一大片柔光。实现可以分成两段：

CSS（氛围光晕载体）：

```css
.carousel::before {
  content: "";
  position: absolute;
  inset: -12%;
  background: var(--active-bg, radial-gradient(900px 500px at 30% 35%, rgba(105, 120, 255, 0.22), rgba(0,0,0,0)));
  filter: blur(26px);
  opacity: 0.95;
  transform: translate3d(0, 0, 0);
  z-index: 0;
  pointer-events: none;
  transition: background 600ms ease;
  will-change: filter, opacity, transform;
}
```

JS（每次切换时，把“当前 slide 的主题渐变”写进容器变量）：

```js
function sync3D() {
  $slides.each((i, el) => {
    const rel = i - realIndex;
    const abs = Math.abs(rel);
    el.style.setProperty("--rel", `${clamp(rel, -2, 2)}`);
    el.style.setProperty("--abs", `${clamp(abs, 0, 2)}`);
  });

  const active = $slides.get(realIndex);
  const bg = getComputedStyle(active).getPropertyValue("--bg").trim();
  if (bg) $carousel[0].style.setProperty("--active-bg", bg);
}
```

**知识点重点**

- 把每张 slide 的“主题色”写成 CSS 变量 `--bg`（inline style），JS 只做“同步当前值”，样式体系更干净。
- 背景光晕用伪元素做，避免影响布局结构。

### 2.2 3D 景深：用 CSS 变量控制每张 slide 的“相对位置”

每张 slide 的 3D 强度不是写死的，而是由 JS 给它打上两个变量：

- `--rel`：相对当前 slide 的位置（-2、-1、0、1、2…）
- `--abs`：相对距离的绝对值（0、1、2…）

然后 CSS 用 `--rel/--abs` 去算 `rotateY / translateZ / scale / opacity`，核心片段如下：

```css
.carousel__slide {
  transform-style: preserve-3d;
  transform:
    perspective(1000px)
    translateZ(calc((1 - min(var(--abs, 2), 2) / 2) * 90px))
    rotateY(calc(var(--rel, 0) * -16deg))
    scale(calc(1 - min(var(--abs, 2), 2) * 0.09));
  opacity: calc(1 - min(var(--abs, 2), 2) * 0.12);
  transition: transform 650ms cubic-bezier(.2,.8,.15,1), opacity 650ms cubic-bezier(.2,.8,.15,1);
}
```

对应 JS 给每张 slide 写入 `--rel/--abs`：

```js
$slides.each((i, el) => {
  const rel = i - realIndex;
  const abs = Math.abs(rel);
  el.style.setProperty("--rel", `${clamp(rel, -2, 2)}`);
  el.style.setProperty("--abs", `${clamp(abs, 0, 2)}`);
});
```

**知识点重点**

- 把“位置关系”交给 JS（简单、确定），把“具体表现”交给 CSS（可调参、可迭代）。
- 3D 动画尽量只动 `transform/opacity`，避免 `filter` 这类高成本特效参与过渡。

## 3. 第三步：建立索引体系与“居中定位”

轮播的切换本质是：给定一个 `index`，算出轨道要平移多少，让该 slide 的中心对齐窗口中心。

### 3.1 用 `realIndex` 管“真实索引（含克隆）”

我们把当前展示位置叫 `realIndex`，它对应的是 `$slides`（轨道上真实存在的 DOM 列表）中的索引。这里是这组“核心状态变量”（精简摘录）：

```js
const $slides = $track.children(".carousel__slide");
let realIndex = 1;
let dragging = false;
let pointerId = null;
let startX = 0;
let startTranslateX = 0;
let currentTranslateX = 0;
let pendingMoveX = 0;
let moveRaf = 0;
let autoplayTimer = null;
let transitionLock = false;
```

### 3.2 居中计算：中心点对齐

核心函数 `getTargetTranslate()`：

- `center = active.offsetLeft + active.offsetWidth / 2`
- `frameWidth = frameEl.getBoundingClientRect().width`
- `translate = -(center - frameWidth / 2)`

对应实现（核心定位公式）：

```js
function getTargetTranslate(index = realIndex) {
  const frameWidth = frameEl.getBoundingClientRect().width;
  const active = $slides.get(index);
  const center = active.offsetLeft + active.offsetWidth / 2;
  return -(center - frameWidth / 2);
}
```

**知识点重点**

- `offsetLeft/offsetWidth` 获取的是布局信息，不要在一帧里反复测量。
- 用“中心点对齐”比“左边缘对齐”更适合卡片轮播（两侧有露出、视觉聚焦）。

## 4. 第四步：无缝循环（Infinite Loop）

无缝循环最常用、也最稳的策略是：**克隆首尾**。

### 4.1 克隆策略：前加最后一张、后加第一张

- 在轨道开头插入“最后一张的 clone”
- 在轨道末尾追加“第一张的 clone”

克隆首尾的实现（无缝循环的基础）：

```js
const $cloneHead = $originalSlides.first().clone(true).attr("data-clone", "true");
const $cloneTail = $originalSlides.last().clone(true).attr("data-clone", "true");
$track.prepend($cloneTail);
$track.append($cloneHead);
```

这样当你：

- 从第 1 张往左：能自然过渡到“最后一张 clone”
- 从最后一张往右：能自然过渡到“第一张 clone”

### 4.2 归位策略：动画结束后瞬移回真实位置

从 clone 过渡完成时会触发 `transitionend`，我们在回调里调用 `normalizeLoop()` 把索引瞬移回真实位置：

- 若 `realIndex === 0`：说明到了头部 clone，归位到 `logicalCount`
- 若 `realIndex === logicalCount + 1`：说明到了尾部 clone，归位到 `1`

实现分两段：

1）监听轨道位移动画结束：

```js
$track.on("transitionend", (e) => {
  if (e.target !== $track[0]) return;
  transitionLock = false;
  normalizeLoop();
});
```

2）在 `normalizeLoop()` 中判断是否需要“从 clone 瞬移归位到真实位置”：

```js
function normalizeLoop() {
  const needsJump = realIndex === 0 || realIndex === logicalCount + 1;
  if (needsJump) {
    $carousel.attr("data-jumping", "true");
    if (realIndex === 0) realIndex = logicalCount;
    else realIndex = 1;
    setTrackTranslate(getTargetTranslate(realIndex), false);
    window.requestAnimationFrame(() => {
      window.requestAnimationFrame(() => {
        $carousel.attr("data-jumping", "false");
      });
    });
  }
  sync3D();
  syncDots();
}
```

### 4.3 处理“回跳抖动”：归位时临时关闭过渡

归位属于“瞬移”，但同时我们还会调用 `sync3D()`，而 slide 的 3D transform 有 transition，容易出现“回跳抖一下”。

做法：归位阶段设置 `data-jumping="true"`，让 `.carousel__track` 与 `.carousel__slide` 都禁用 transition；两帧后再恢复。

对应 CSS：

```css
.carousel[data-jumping="true"] .carousel__track,
.carousel[data-jumping="true"] .carousel__slide {
  transition: none;
}
```

**知识点重点**

- “动画过渡到 clone + 动画结束后瞬移归位”是无缝循环的经典解法。
- 抖动通常不是位移本身，而是“归位瞬间触发了其它带 transition 的属性变化”。解决思路就是：归位阶段统一禁用相关 transition。

## 5. 第五步：控制系统（按钮、指示点、键盘）

### 5.1 统一入口：`goToReal()` / `go()`

- `goToReal(index)`：跳到指定真实索引（含 0 与 logicalCount+1 的边界）
- `go(dir)`：相对移动一张

对应核心切换入口（所有切换逻辑都收口在这里）：

```js
function goToReal(nextRealIndex, { animate = true, userInitiated = false } = {}) {
  if (transitionLock && animate) return;
  const capped = clamp(nextRealIndex, 0, logicalCount + 1);
  realIndex = capped;
  $carousel.attr("data-dragging", "false");
  const targetTranslate = getTargetTranslate(realIndex);
  const shouldAnimate = animate && Math.abs(targetTranslate - currentTranslateX) > 0.5;
  transitionLock = shouldAnimate;
  setTrackTranslate(targetTranslate, shouldAnimate);
  sync3D();
  syncDots();
  if (userInitiated) restartProgress();
}

function go(dir, userInitiated = false) {
  goToReal(realIndex + dir, { animate: true, userInitiated });
}
```

### 5.2 指示点：建立并同步状态

- `buildDots()`：初始化指示点按钮

```js
function buildDots() {
  const frag = document.createDocumentFragment();
  for (let i = 1; i <= logicalCount; i++) {
    const btn = document.createElement("button");
    btn.type = "button";
    btn.className = "carousel__dotbtn";
    btn.setAttribute("aria-label", `跳转到第 ${i} 张`);
    btn.setAttribute("aria-current", i === 1 ? "true" : "false");
    btn.dataset.index = String(i);
    frag.appendChild(btn);
  }
  $dots[0].appendChild(frag);
}
```

- `syncDots()`：根据 `realIndex` 映射到逻辑索引（1..N），更新 `aria-current`

```js
function logicalFromReal(index) {
  if (index === 0) return logicalCount;
  if (index === logicalCount + 1) return 1;
  return index;
}

function syncDots() {
  const currentLogical = logicalFromReal(realIndex);
  $dots.children("button").each((i, el) => {
    el.setAttribute("aria-current", i + 1 === currentLogical ? "true" : "false");
  });
}
```

- 点击事件委托（指示点容器上统一处理）

```js
$dots.on("click", "button", onDotClick);
```

### 5.3 键盘与按钮

- 键盘左右键：

```js
function onKeydown(e) {
  if (e.key === "ArrowLeft") {
    e.preventDefault();
    go(-1, true);
  } else if (e.key === "ArrowRight") {
    e.preventDefault();
    go(1, true);
  }
}
```

- 上一张/下一张按钮：

```js
function onActionClick(e) {
  const action = $(e.currentTarget).data("action");
  if (action === "prev") go(-1, true);
  if (action === "next") go(1, true);
}
```

**知识点重点**

- 把所有“切换”都收敛到 `goToReal()`，可减少分支 bug（例如进度条重置、背景同步、3D 同步就不会漏）。
- 指示点用 `aria-current` 表示当前项，对可访问性更友好。

## 6. 第六步：拖拽/滑动（Pointer Events）

拖拽交互的关键是“按下记录起点、移动实时预览、松开判断阈值切换或回弹”：

- `pointerdown`：记录 `startX/startTranslateX`，并暂停自动播放
- `pointermove`：计算 `dx`，把轨道跟着手指走
- `pointerup`：根据 `dx` 是否超过阈值，决定 `go(±1)` 还是回到当前张

实现（精简版，保留关键逻辑）：

```js
function onPointerDown(ev) {
  if (ev.button !== undefined && ev.button !== 0) return;
  if (transitionLock) return;
  pointerId = ev.pointerId ?? null;
  setDragging(true);
  stopAutoplay();
  if (pointerId != null) frameEl.setPointerCapture?.(pointerId);

  startX = ev.clientX;
  pendingMoveX = startX;
  startTranslateX = currentTranslateX;
}

function flushDragMove() {
  moveRaf = 0;
  const dx = pendingMoveX - startX;
  setTrackTranslate(startTranslateX + dx, false);
}

function onPointerMove(ev) {
  if (!dragging) return;
  if (pointerId != null && ev.pointerId != null && ev.pointerId !== pointerId) return;
  pendingMoveX = ev.clientX;
  if (!moveRaf) moveRaf = window.requestAnimationFrame(flushDragMove);
}

function onPointerUp(ev) {
  if (!dragging) return;
  if (pointerId != null && ev.pointerId != null && ev.pointerId !== pointerId) return;

  setDragging(false);
  if (moveRaf) {
    window.cancelAnimationFrame(moveRaf);
    moveRaf = 0;
    pendingMoveX = ev.clientX;
    flushDragMove();
  }
  if (pointerId != null) frameEl.releasePointerCapture?.(pointerId);
  pointerId = null;

  const active = $slides.get(realIndex);
  const dx = currentTranslateX - startTranslateX;
  const threshold = Math.max(42, active.offsetWidth * 0.12);
  if (dx > threshold) go(-1, true);
  else if (dx < -threshold) go(1, true);
  else goToReal(realIndex, { animate: true, userInitiated: true });

  startAutoplay();
}
```

为了让拖拽更流畅，我们对 `pointermove` 做了 `requestAnimationFrame` 节流：

- `pointermove` 只记录最新坐标，真正的 DOM 更新在下一帧的 `flushDragMove()` 执行（避免高频事件直接写 DOM）

事件监听使用原生 `addEventListener` + `{ passive: true }`，减少主线程阻塞风险：

```js
frameEl.addEventListener("pointerdown", onPointerDown, { passive: true });
window.addEventListener("pointermove", onPointerMove, { passive: true });
window.addEventListener("pointerup", onPointerUp, { passive: true });
window.addEventListener("pointercancel", onPointerUp, { passive: true });
```

**知识点重点**

- Pointer Events 统一鼠标与触摸，比单独处理 `mouse/touch` 更简单。
- 高频事件（move/scroll）避免每次都写 DOM，`rAF` 节流是最通用的手段。
- `touch-action: pan-y` 允许纵向滚动，不会把页面滚动完全锁死：

```css
.carousel__frame {
  touch-action: pan-y;
}
```

## 7. 第七步：自动播放 + 进度条（并与暂停同步）

### 7.1 自动播放：用 interval 统一调度

- `startAutoplay()`：启动 interval，每次 `go(1)` 并重置进度条
- `stopAutoplay()`：清理 interval，并标记暂停状态

对应实现：

```js
function stopAutoplay() {
  clearAutoplayTimer();
  $carousel.attr("data-paused", "true");
}

function startAutoplay() {
  if (autoplayMs <= 0) {
    stopAutoplay();
    return;
  }
  clearAutoplayTimer();
  $carousel.attr("data-paused", "false");
  autoplayTimer = setInterval(() => {
    go(1, false);
    restartProgress();
  }, autoplayMs);
  restartProgress();
}
```

### 7.2 进度条：用纯 CSS 动画，JS 只负责重置/暂停

进度条是 `.carousel__bar` 的 `width: 0% -> 100%` 动画：

CSS 动画（进度条增长 + 暂停）：

```css
.carousel__bar {
  height: 100%;
  width: 0%;
  animation: progress var(--autoplay-ms, 5200ms) linear infinite;
}

.carousel[data-paused="true"] .carousel__bar {
  animation-play-state: paused;
}

@keyframes progress {
  from { width: 0%; }
  to { width: 100%; }
}
```

JS 重置（让进度条从 0 重新开始）：

```js
function restartProgress() {
  if (autoplayMs <= 0) return;
  $bar[0].style.animation = "none";
  void $bar[0].offsetHeight;
  $bar[0].style.animation = "";
}
```

### 7.3 暂停策略：hover 暂停 + 切后台暂停

```js
$carousel.on("mouseenter", stopAutoplay);
$carousel.on("mouseleave", startAutoplay);

document.addEventListener("visibilitychange", () => {
  if (document.hidden) stopAutoplay();
  else startAutoplay();
}, { passive: true });
```

**知识点重点**

- 进度条最好别用 JS 每帧改宽度（昂贵且容易不准），用 CSS 动画最省心。
- “暂停状态”不要只停 timer，最好同时让进度条动画也停（用户会感觉更一致）。

## 8. 第八步：可访问性与“减少动态效果”

### 8.1 减少动态效果

当用户系统偏好 `prefers-reduced-motion: reduce` 时：

- JS 把 `autoplayMs` 设为 0，从逻辑上关闭自动播放
- CSS 把 track/slide/背景过渡关掉，进度条直接显示满格

对应实现：

JS（根据偏好关闭自动播放，并同步给进度条变量）：

```js
const prefersReducedMotion = window.matchMedia?.("(prefers-reduced-motion: reduce)")?.matches ?? false;
const autoplayMs = prefersReducedMotion ? 0 : 5200;
$carousel[0].style.setProperty("--autoplay-ms", `${autoplayMs}ms`);
$carousel.attr("data-paused", autoplayMs > 0 ? "false" : "true");
```

CSS（减少动态时，关掉过渡与进度条动画）：

```css
@media (prefers-reduced-motion: reduce) {
  .carousel::before { transition: none; }
  .carousel__track { transition: none; }
  .carousel__slide { transition: none; }
  .carousel__bar { animation: none; width: 100%; }
}
```

### 8.2 ARIA 与键盘

- 轮播容器的 ARIA（roledescription + label + 可聚焦）：

```html
<section class="carousel" id="coolCarousel" aria-roledescription="carousel" aria-label="炫酷轮播图" tabindex="0">
  ...
</section>
```

- 指示点的 ARIA（给每个按钮明确 label，并用 `aria-current` 表示当前项）：

```js
btn.setAttribute("aria-label", `跳转到第 ${i} 张`);
btn.setAttribute("aria-current", i === 1 ? "true" : "false");
```

**知识点重点**

- 轮播属于强动画组件，尊重 `prefers-reduced-motion` 是基本要求。
- `aria-current` 是表达“当前指示点”的简单、通用手段。

## 9. 常见坑与修复思路（本项目踩过的）

### 9.1 自动播放“不自动切”

如果用 `focusin/focusout` 来控制暂停，很容易出现页面加载后一直处于“聚焦暂停”状态，导致你感觉自动播放坏了。现在我们改成 hover 暂停 + `visibilitychange` 暂停。

这部分的实现代码在第 7 节（自动播放 + 暂停策略）的代码块里。

### 9.2 无缝循环回跳“抖动”

症状：最后一张跳第一张时，轨道归位是瞬移的，但 slide 的 3D transition 仍在，导致视觉抖动。

修复：归位阶段加 `data-jumping` 禁用 transition，两帧后恢复。对应实现见第 4 节（归位防抖）的代码块。

### 9.3 性能卡顿

轮播卡顿一般来自两类：

- **高成本特效**（大面积 blur/backdrop-filter/filter 参与动画）
- **高频事件直接写 DOM**（pointermove 每次都改 transform、同时还频繁读布局）

本项目的策略是：

- 让动画尽量只涉及 `transform/opacity`
- 拖拽 move 使用 `requestAnimationFrame` 节流

对应实现见第 6 节（拖拽 rAF 节流）与第 2 节（3D transform 仅动 transform/opacity）的代码块。

## 10. 知识点总结（可背诵版）

- **轨道模型**：窗口裁切 + 轨道平移，切换就是改 `translate3d`。
- **定位公式**：`translate = -(slideCenter - frameCenter)`，用中心点对齐做卡片轮播更自然。
- **无缝循环**：首尾 clone + `transitionend` 瞬移归位；归位阶段注意禁用其他 transition 防抖。
- **3D 炫酷**：JS 只产出 `--rel/--abs`，CSS 负责视觉表达；背景光晕用 `--active-bg` 同步。
- **拖拽手感**：Pointer Events + 阈值判断；move 用 rAF 节流，不要每次事件都写 DOM。
- **自动播放一致性**：timer 与进度条同步；暂停不仅停 timer，也要暂停进度条动画；切后台暂停。
- **可访问性**：`prefers-reduced-motion` 必须支持；指示点用 `aria-current`；键盘左右键可用。
